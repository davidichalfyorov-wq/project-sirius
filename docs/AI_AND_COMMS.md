# Project Sirius — Ambient AI и переговоры пилотов (GCS)

Дата: 2026-06-09. Статус: реализовано, покрыто тестами.

## 1. Обзор

Две связанные системы:

1. **Ambient AI** — населённый космос: торговцы, патрули, бродячий трафик
   с поведением по типу encounter-зоны, формациями, репопуляцией,
   бегством под огнём и взаимопомощью фракций.
2. **GCS chatter** — радиопереговоры пилотов, собранные из оригинальных
   аудио-сегментов Freelancer (`gcs_*`-клипы в voice-паках Discovery),
   с ванильной грамматикой фраз и позывными.

Обе работают только на сервере; клиент получает готовые последовательности.

## 2. Ambient AI

### Файлы
| Файл | Роль |
|---|---|
| `src/LibreLancer/Server/Components/SAmbientBehaviorComponent.cs` | Поведение одного корабля (wander / patrol_path / trade) + бегство |
| `src/LibreLancer/Server/ServerWorld.cs` | Спавн по зонам, формации, репопуляция, `RequestAssistance` |
| `src/LibreLancer/World/EncounterHandler.cs`, `EncounterInfo.cs` | Прокидывают `behavior` из `[EncounterFormation]` |

### Поведения (из `behavior =` encounter-файла)
- **trade** — выбирает стыкуемую цель (база / jump-гейт / кольцо
  трейдлейна) в 3–120 км, летит на крейсере, стыкуется. Стыковка
  деспавнит корабль («улетел»); кольцо лейна физически перевозит его
  через систему. Таймаут цели 240 с.
- **patrol_path** — петля из 5 точек вокруг зоны на полной тяге.
- **wander** — точки внутри зоны, тяга 0.9, крейсер разрешён.

### Взаимосвязанность
- **Формации**: группы (кроме торговцев) летят строем за лидером;
  лидер, вошедший в трейдлейн, затягивает ведомых (механика движка).
  Осиротевшие ведомые сами покидают строй и продолжают по поведению.
- **Бегство**: торговец с хуллом < 65 % под прицелом бросает дела,
  уходит на крейсере прочь 45 с и зовёт о помощи в эфир.
- **Взаимопомощь** (`ServerWorld.RequestAssistance`): первый контакт
  с врагом созывает патрули/бродяг той же фракции в радиусе 7 км —
  свободные (без directive, без боя, не торговцы) получают `Attack`.
- **Бой**: как только у NPC появляется цель, ambient-компонент молчит —
  работает штатный state-graph (buzz/trail/evade, тайминги пилота).
- **Репопуляция**: каждые 40–60 с система дозаполняется до бюджета
  36 кораблей — взамен улетевших и сбитых.

## 3. Переговоры пилотов (GCS)

### Файлы
| Файл | Роль |
|---|---|
| `src/LibreLancer/Server/Comms/GcsVoiceLines.cs` | Грамматика фраз: событие → последовательность сегментов |
| `src/LibreLancer/Server/Comms/SChatterComponent.cs` | Голос + постоянный позывной NPC, кулдаун 18 с |
| `src/LibreLancer/Server/Comms/ChatterDispatcher.cs` | Глобальный эфир: 1 фраза / 5 с, слышимость 12 км, ATC-ответы |
| `src/LibreLancer/Net/Protocol/IClientPlayer.cs` (`Chatter`) | RPC: массив `NetDlgLine` = последовательность клипов |
| `src/LibreLancer/Client/CGameSession.Space.cs` (`RunChatter`) | Клиент: проигрывает цепочку клип-за-клипом |

### Грамматика (как в оригинале)
Источник истины — сами данные: `voices_space_male/female.ini`
(`[Voice]` + `[Sound] msg = <сегмент>`), `faction_prop.ini`
(`msg_id_prefix`, `formation_desig`, `voice`), voice-паки `.utf`
(клипы по FLHash от имени сегмента).

- Позывной: `gcs_refer_faction_<fac>_short` →
  `gcs_refer_formationdesig_NN` → `gcs_misc_number_N` →
  `gcs_misc_number_N` (формы середины фразы, без суффикса).
- Суффикс `+` = восходящая интонация (фраза продолжается),
  `-` = терминальный клип. Запрос дока заканчивается на
  `gcs_dockrequest_request_NN+` — как вопрос; ответ ATC
  (`atc_leg_m01/f01`) — `gcs_dockrequest_granted_01-`.
- `formation_desig` IDS (197808 = первый) → номер клипа `NN`.
- Пермутации (`_01-`, `_02-`, …) выбираются случайно **только из
  клипов, которые есть у конкретного голоса**. Если хоть один сегмент
  фразы отсутствует — NPC молчит; неверная последовательность не
  играется никогда.

### События → фразы
| Событие | Последовательность |
|---|---|
| Запрос дока (торговец) | позывной + `dockrequest_request_NN+`, через 3–5 с ATC: `granted_01-` |
| Первый контакт | ведомый: `combat_announce_enemysighted_NN-`; лидер звена: `combat_order_engage_NN-` |
| Получает урон | `combat_taking_damage_NN-` |
| Попадает по цели | `combat_inflicting_damage_NN-` |
| Бегство | `combat_fleereason_damaged_NN-` |
| Смерть | `combat_scream_NN-` (без кулдауна) |
| Конец боя (8 с тишины) | `combat_announce_allclear_NN-` |

### Дисциплина эфира
Глобально не чаще одной фразы в 5 с; каждый корабль — раз в 18 с;
слышимость 12 км от игрока; смерть игнорирует личный кулдаун.

## 4. Тесты

- `src/LibreLancer.Tests/GcsChatterTests.cs` — 7 xUnit-тестов:
  порядок позывного, терминальность боевых клипов, «каждый сегмент
  существует у голоса», тишина при нехватке клипа, диапазоны позывных,
  выбор leg/ill-варианта. Весь набор: **269/269 passed**.
- `Output/verify_gcs_sequences.py` — проверка по «живым» данным
  Discovery: 33 голоса, 15 970 сегментов, 99 фракций, 259 пар
  «фракция×голос». Результат: боевые фразы собираются для **100 %**
  пар; запрос дока — для 223 (остальные 36 — голоса без клипа имени
  фракции, корректно молчат); `granted` подтверждён у ATC-голосов.

## 5. Автотест-режим (SIRIUS_AUTOPLAY)

`SIRIUS_AUTOPLAY=1` — dev/CI-режим (`src/LibreLancer/SiriusAutoplay.cs`):
меню само начинает новую игру (`LuaMenu`), база автоматически взлетает
через 4 с (`RoomGameplay`), а первый репоп-тик после входа игрока в
космос принудительно вызывает один запрос дока у ближайшего NPC
(`ServerWorld`, лог `Autoplay: forced chatter test`). Это позволяет
headless-прогоном проверить весь путь: сборка фразы → RPC → клиентская
цепочка аудио-клипов. Без переменной окружения режим полностью неактивен.

Серверные фразы логируются: `Chatter: <ник> [<голос>]: seg1 -> seg2 …`;
клиент логирует каждый сыгранный клип: `Chatter: playing <голос> 0x<hash>`.

## 6. Ограничения (честно)

- В voice-паках Discovery нет клипов имён баз/систем (в отличие от
  чистой ванилы), поэтому фразы «en route to …» с именем базы
  непроизносимы на этих данных и не используются.
- Сканирование груза, vanilla-«buzz player» и комм-окно с головой для
  ambient-фраз не включены (механика подготовлена, см. `OpenComm`).
