# CI / автоматическая проверка для ChatGPT (и любого агента без GPU)

Назначение: дать агенту, который **не может собрать и запустить игру у себя** (нет .NET, нет GPU, нет прав
ставить пакеты), способ всё-таки проверять свои патчи. Прямо в песочнице ChatGPT GPU-игру запустить нельзя
физически — мостом служит **CI на GitHub Actions**. Настроено два уровня.

---

## Уровень 1 — СБОРКА (compile + шейдеры). Автоматически, без настройки.

Файл: `.github/workflows/build.yml`. Триггер: каждый `push` (кроме правок только данных/доков) и каждый PR.

Что делает (на бесплатном GitHub-hosted раннере, **без GPU и без 7 ГБ LFS-данных**):
1. Checkout кода (LFS-данные Discovery пропускаются — `GIT_LFS_SKIP_SMUDGE=1`, квота не тратится).
2. Ставит .NET 10 + cmake/make.
3. Скачивает закреплённый в `build.config` `dxc` (тот же, которым собирает проект).
4. `dotnet build src/lancer/lancer.csproj -c Release` — компилирует C# **И HLSL-шейдеры** (через `Shader.targets`).

**Зелёная галочка = твой патч собирается, шейдеры компилируются, 0 ошибок.** Красная = смотри лог шага
«Build game + shaders», там точная ошибка C#/SH. Это закрывает главную слепую зону: компилируется ли код.

Как читать результат (тебе, ChatGPT):
- После пуша — статус на странице коммита/PR на GitHub (значок ✓/✗), либо вкладка **Actions**.
- Публичный репозиторий → статус и логи доступны без авторизации; владелец может прислать ссылку на прогон,
  или ты можешь запросить лог конкретного шага.

> ВАЖНО: build-гейт проверяет ТОЛЬКО сборку, не внешний вид. Рендеринг волюметрики требует GPU (Уровень 2).

---

## Уровень 2 — РЕНДЕР (визуальная проверка). Нужна одноразовая настройка владельцем.

Файлы: `.github/workflows/render-selfhosted.yml` + `scripts/ci_render.sh`. GitHub-hosted раннеры **без GPU**,
поэтому рендер идёт на **self-hosted раннере** — машине владельца (Linux + RTX), где игра и так собирается и
рисуется. На ней есть .NET, GPU, dxc и данные Discovery (по `freelancer_path`).

Одноразовая настройка владельцем (5 минут):
1. Repo → **Settings → Actions → Runners → New self-hosted runner** (Linux x64). Добавить метку **`gpu`**.
2. Выполнить показанные `./config.sh …` и запустить `./run.sh` (или поставить службой) **под тем же
   пользователем**, у которого в `~/.config/librelancer.ini` прописан `freelancer_path`.

После этого:
- Запуск вручную: **Actions → «Render (self-hosted GPU)» → Run workflow** (можно задать позы/режим), ИЛИ
  пуш в ветку `ci-render`.
- Воркфлоу собирает игру и снимает стандартные позы (ближняя/дальняя Badlands), кладёт PNG + `montage.png`
  в **артефакты прогона** (вкладка Actions → конкретный run → Artifacts).
- `ci_render.sh` параметризован: `CI_POSES="name:x,y,z,yaw …"`, `CI_LIVE=1|0`, `SIRIUS_RENDERER=vulkan|gl`.

Скрипт `scripts/ci_render.sh` самодостаточен и переносим (сам находит корень проекта) — владелец может
запускать его и вручную: `scripts/ci_render.sh` → результат в `Output/ci_render/`.

---

## Разделение труда (честно)

| Что | Кто/где | Автоматизм |
|-----|---------|------------|
| Компилируется ли патч (C# + шейдеры) | GitHub-hosted CI | автоматически на каждый push |
| Как это выглядит (волюметрика, кадры) | self-hosted GPU-раннер ИЛИ владелец локально | по запросу / пуш в `ci-render` |
| Запуск игры в песочнице ChatGPT | невозможно (нет GPU/.NET) | — |

Рекомендуемый цикл для ChatGPT: пиши патч → пуш → дождись зелёного **build**-гейта (правь, пока не соберётся) →
запроси у владельца (или дёрни render-воркфлоу) визуальную проверку → итерируй по скриншотам. Не утверждай
«не могу проверить»: сборку теперь проверяет CI автоматически.
